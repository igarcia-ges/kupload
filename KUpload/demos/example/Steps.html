Getting started
The following example will show you how to easily implement the Kaltura Simple Uploader (KSU) on to your web page.

The KSU has a flash object loaded in order to enable the browsing and selecting of content to upload.
We can find it in the code where it says: "--embed flash object--"

The first step in understanding the code behind this example is to know when the SWF flash object is loaded.
Look for the following handler, in the KSU handlers section, where it says: delegate.readyHandler
That's where we know if the flash object was loaded or not.
You can see that once the object was loaded we throw an alert message.
Click on it to jump to Step 1 - Browsing for content.
Step 1 - Browsing for content
Great! now click on the button saying: "Step 1 - Browse for content" and select the desired content to upload.
Step 2 - Uploading content
Great! We have now selected the content to upload. View the results section to view the total size of content we selected.

How do we know that ?
Easy! We can find the handler who takes care of it in the code, in the KSU handlers section where it says: "delegate.selectHandler"

There are lots of other actions you might like to take before uploading, such as: Add tags, Add titles etc.
To better understand how to do it, please go to "KSU Website Integration Guide" on: http://www.kaltura.org/kaltura-simple-uploader-ksu-website-integration-guide

What's next ? Let's upload the selected content!

Click on the "Step 2 - Upload selected content" button.

Step 2.1 - Uploading progress & Done uploading
Now we are uploading the content to the Kaltura network.
We can see in the results pane the upload progress for each file selected for upload:

How do we know what is the content upload progression ?
Easy! We can find the handler who takes care of it in the code, in the KSU handlers section where it says: "delegate.progressHandler"

When each file done uploading, we'll see in the results pane the name of the file done.
Also, we'll see in the result pane amessage when all files are done uploading.

And how do we know that ?
Easy! We can find the handler who takes care of it in the code, in the KSU handlers section where it says: "delegate.singleUploadCompleteHandler" & "delegate.allUploadsCompleteHandler"
Step 3 - Add entries
Now, let's click on the "Step 3 - Add entries" button to add the uploaded files as entries in our KMC.

Once clicked, the uploaded files are being set as entries in the KMC (http://www.kaltura.com/kmc).
The function that start this procedure is called: "addEntries()".

Once the entries are added, we'll see a message in the results pane.
How do we know that entries are successfully added to the KMC ?
Easy! We can find the handler who takes care of it in the code, in the KSU handlers section where it says: "delegate.entriesAddedHandler"

Step 4 - Get files
Now, once the files are set as entries, let's get the list of files added.

How do we do that ?
Let's click on the: "Step 4 - Get Files" button.

Once clicked, we'll see the list of files in the results pane.
The function responsible for that action called: "getFiles()"

Step 5 - Actions
That's it!

We have now completed the upload process with the KSU.

Best of luck implementing the KSU in your site.
